📅 4-day plan
Sprint 1 · Day 1 morning Database + skeleton
Everything else depends on this. Do it first.
  • 1Init PostgreSQL schema — users, roles, devices, sessions, files, shares, audit_logs
  • 2Spin up Redis (Docker Compose) — OTP + session + revocation cache
  • 3Spin up MinIO (Docker Compose) — create files bucket with versioning on
  • 4FastAPI project scaffold — router files for auth / files / shares / audit
  • 5Health-check endpoint GET /health → verify all services respond
~3 hrs
Sprint 2 · Day 1 afternoon Auth pipeline
Nothing works without login. Build the full auth chain end-to-end.
  • 1POST /auth/login — bcrypt verify, OTP send to email
  • 2POST /auth/verify-otp — check Redis OTP, capture device fingerprint
  • 3Issue RS256 JWT (15-min) + opaque refresh token (8-hr, Redis-stored)
  • 4JWT middleware — verify on every protected route, check revoked-JTI blacklist
  • 5RBAC role decorator — @require_role("admin") guards on routes
  • 6POST /auth/logout — blacklist JTI in Redis
~4 hrs
Sprint 3 · Day 2 morning File upload + crypto engine
The core value prop. Envelope encryption in-process, never plaintext to disk.
  • 1Crypto helper — generate_dek(), aes_gcm_encrypt(dek, file_bytes)
  • 2Key wrap — rsa_oaep_wrap(kek_pub, dek) → store wrapped key + IV in DB
  • 3POST /files/upload — multipart → encrypt → PUT to MinIO → INSERT files record
  • 4SHA-256 digest computed at upload; stored in files.sha256
  • 5GET /files/{id} — fetch + unwrap DEK + decrypt stream → verify hash → serve
  • 6Audit log write on every upload + download event
~4 hrs
Sprint 4 · Day 2 afternoon Share system + policy engine
This is what the demo hinges on. Build the full share-and-block loop.
  • 1POST /shares — generate token, hash-store in DB, set Redis TTL, expiry + dl_limit
  • 2Device fingerprint lock — store fingerprint on first access, deny on mismatch
  • 3GET /dl/{token} — validate token hash, decrement dl_remaining, deliver file
  • 4DELETE /shares/{id} — instant revoke: Redis DEL + DB status=REVOKED
  • 5ABAC basic check — risk_score + device_trusted + time_window conditions
  • 6Watermark injection — embed user/token/timestamp into served PDF/image
~4 hrs
Sprint 5 · Day 3 Risk engine + frontend + demo
Make it visible and demonstrable. Risk scoring + React UI + drill rehearsal.
  • 1Risk scorer — signal weights, cumulative score, MFA/deny thresholds (30/60/85)
  • 2Alert table + Prometheus metric stubs (login_failures, policy_denials, dl_rate)
  • 3React: Login → Dashboard → Upload → Share centre (minimal Tailwind)
  • 4Admin panel: live audit log table, share revoke button, risk score badge
  • 5Full demo dry-run: upload → share → vendor opens → forward blocked → admin traces
  • 6Docker Compose final — one command boots everything for judges
~6 hrs
Day 4 (April 30): Buffer — fix blockers, polish demo, write a one-page README. Don't add features on the last day.
